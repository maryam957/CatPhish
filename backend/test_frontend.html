POST /api/auth/register

POST
Step 1: Register a new user. In demo mode the response includes _demoVerifyToken — copy it for Step 2. Password must be 8+ chars with 3 of: uppercase, lowercase, digit, special char.
Min 8 chars. Need 3 of: uppercase, lowercase, digit, special char (e.g. Test1234!)

POST /api/auth/verify-email

POST
Step 2: Paste the _demoVerifyToken from the register response below. In production this would come via email.
Alternative (Demo mode only): Use "Demo Activate" in the sidebar to skip verification entirely.

POST /api/auth/login

POST
Step 3: Login. On success an access token (15 min) and CSRF token are stored in memory for subsequent requests. A refresh token is set as an httpOnly cookie.

GET /api/auth/me

GET
Requires a valid access token. Returns the logged-in user's id, email, and role.

POST /api/auth/change-password

POST
Requires access token + CSRF token (automatically sent from session). Requires re-confirmation of the current password.

POST /api/auth/refresh

POST
Uses the httpOnly refresh token cookie (set automatically during login) to issue a new access token. No body needed.

POST /api/auth/logout

POST
Revokes the refresh token and CSRF tokens. Clears the httpOnly cookie.

POST /api/reports

POST
Public endpoint (no auth required). Reports a phishing URL by hash prefix. Nonce and timestamp prevent replay attacks.

GET /api/reports/admin

GET
Requires admin role. Login as an admin user first, or use "Make Admin (Demo)" in the sidebar.

GET /api/threat-intel/updates

GET
Public endpoint. Returns the current threat hash-prefix database. No auth required.

POST /api/threat-intel/entries

POST
Requires admin role + access token + CSRF token.

DELETE /api/threat-intel/entries/:prefix

DELETE
Requires admin role.

GET /api/threat-intel/audit

GET
Requires admin role.

POST /api/url-analysis/report

POST
Public endpoint. Extension sends URL hash prefix + client risk score. Server aggregates and returns a signed verdict.

GET /api/url-analysis/stats

GET
Requires admin role.

POST /api/sessions/register

POST
Extension registers its HMAC-SHA256 key. The sessionId must be 16-64 hex chars. The key JWK must be a valid HMAC-SHA256 JWK with at least 32 chars of base64url key material.

POST /api/auth/demo-activate

POST
Demo mode only! Activates a user account without email verification. Use this instead of copying the verify token.

🔧 Make Admin (Demo Utility)

UTILITY
Important: The backend has no built-in "make admin" endpoint — roles are assigned server-side only. To test admin endpoints you need to:

1. Register + activate a user
2. Stop the server
3. Start it with the user's userId and manually set role in memory (or use the helper below to show instructions)
Quick workaround for testing: Start your server with a seeded admin user by running the quick-start script (see below). Or restart the server process and register as admin@catphish.local — the server auto-seeds this account in demo mode after the fix.
